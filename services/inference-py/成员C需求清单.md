# 📄 需求文档：AI 语义审查与模型中台 (Python 模块)

## 1. 任务概述

利用 Qwen 系列大模型，构建论文语义审查引擎。该引擎需具备对学术文本的**纠错、润色建议、逻辑一致性检测**及**文献真伪验证**能力。

## 2. 技术栈要求(可修改)

* **模型选型：** Qwen（通过 DashScope API 或私有化 vLLM 部署）。
* **框架：** FastAPI, PyTorch, LangGraph (处理多步推理逻辑)。
* **向量库：** Milvus / Pinecone (用于参考文献真伪比对)。
* **性能优化：** TensorRT-LLM 加速。

---

## 3. 核心功能模块与执行清单

### 模块一：学术语境高风险项检测 (Semantic Error Detection)

* **需求描述：** 识别不仅限于错别字，还包括学术语境下的不当用词。
* **Qwen 任务：**
* **纠错 (Typo)：** 识别形近字错误（如“黏度”误写为“粘度”、“分量”误写为“份量”）。
* **标点规范：** 检测中文段落中的半角标点、不规范的数学符号。
* **学术风格检查：** 识别口语化（如“听说”、“特别好”）并建议修改为学术化词汇（如“据报道”、“具有显著优势”）。

* **交付物：** 一个高并发推理接口，支持流式输出错误坐标与修改建议。

### 模块二：参考文献真实性校验 (Fact-Checking RAG)

* **需求描述：** 防止 AI 幻觉导致的“虚假文献”或作者信息错误。
* **执行逻辑：**

1. 提取文末参考文献条目。
2. **检索：** 将条目转化为 Embedding，在学术数据库索引（如 arXiv, Crossref 镜像）中检索。
3. **比对 (Qwen-Long)：** 使用 Qwen 比较“论文引用的信息”与“检索到的真实信息”是否一致（年份、卷号、标题）。

* **交付物：** 文献真实性分析报告。

### 模块三：长文本逻辑一致性扫描 (Logic Consistency)

* **需求描述：** 确保文章前后表述一致。
* **Qwen 任务：**
* **摘要-结论对齐：** 检测摘要中承诺的研究内容在结论中是否得到体现。
* **术语一致性：** 监控全篇术语。例如，前文使用“卷积神经网络”，后文突然切换为“CNN”且未定义缩写。

## 4. Prompt 工程设计 (核心机密)

成员 C 需在 LangGraph 中维护以下核心 Prompt 策略：

```markdown
# Role
你是一位精通中文学术规范的审稿专家，专门负责检测论文中的细微语义错误。

# Task
输入为一段论文正文。请输出以下 JSON：
{
  "issues": [
    {
      "type": "TYPO | PUNCTUATION | STYLE",
      "original": "错误文本",
      "suggested": "修改后的文本",
      "reason": "解释为什么此处不合规",
      "severity": 1-5
    }
  ]
}

# Knowledge Base (Qwen Specific)
- 遵循《通用规范汉字表》
- 遵循 GB/T 15834 标点符号用法
- 识别理工科、人文社科的不同用词习惯

```
